# Suggested Implementation Sequence

## Sequence A — Stabilise the foundations

1. Build the work item state store.
2. Create the provider/principal configuration library.
3. Build the regression corpus.
4. Refactor CE Document Mapper extraction into a reusable service.
5. Add the human review queue.
6. Add monitoring, runbooks and release controls.

## Sequence B — Connect core systems

1. Automate source file storage and Box metadata.
2. Add Outlook/Graph event intake.
3. Add duplicate/historical search.
4. Add EVA/Sentry assisted import.
5. Pilot with a small number of stable providers.
6. Expand provider coverage only after test results are stable.

## Sequence C — Add staff-facing workflow tools

1. Add image quality/coverage assistant.
2. Add missing-file chaser drafting.
3. Add engineer pack generation.
4. Add estimate/ABP review pack.
5. Add knowledge base assistant.

## Sequence D — Expand external access and intelligence

1. Add customer portal for upload/status.
2. Add selected insurer/work-provider integrations.
3. Add partner API for approved partners.
4. Add analytics dashboards.
5. Add data warehouse and archival.
6. Pilot risk indicators only after governance sign-off.
7. Add predictive scheduling once historical data is reliable.
