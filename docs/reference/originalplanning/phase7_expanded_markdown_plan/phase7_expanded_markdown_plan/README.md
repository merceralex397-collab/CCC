# Phase 7 Expanded Plan Pack

This pack expands `phase_additional.md` into separate, step-by-step markdown plans. It also adds further opportunities drawn from the wider Collision Engineers project context.

## How to read this pack

1. Start with `00_plan_review_and_crosscutting_recommendations.md`.
2. Read `02_prioritised_roadmap.md` before acting on any individual plan.
3. Use `original_items/` for the nine Phase 7 items already listed in the plan file.
4. Use `additional_items/` for further candidate additions identified from the project context.
5. Use `appendices/` for governance, acceptance criteria, source notes and decision records.

## Key position

The Phase 7 ideas are valid, but they should not be treated as a single build phase. They split into four different categories:

- **Near-term foundations:** state store, review queue, provider/principal configuration, test corpus, monitoring, governance.
- **Medium-term workflow extensions:** self-service portal, engineer pack generator, image quality assistant, estimate/ABP review, communications automation.
- **Integration layer:** EVA/Sentry API, insurer/work-provider APIs, Audatex-approved integrations, partner API.
- **Longer-term intelligence:** predictive scheduling, analytics, data warehouse, risk indicators, knowledge base.

## Immediate caution

Do not start with fraud scoring, automated expert conclusions, or external partner APIs. The system should first establish clean work item state, reliable intake, Box/file traceability, validation, human review and EVA integration. The higher-risk ideas become much safer once those foundations exist.
