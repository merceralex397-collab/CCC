# Finance Billing Workspace

## Purpose

This workspace owns all planning for invoice handling, fee-note generation, payment status tracking, chaser metadata, approvals, and payment automation option papers.

## Scope Rules

- Payment automation and autonomous chasers are long-range planned and require governance approval (option paper first).
- Current payment and fee-note evidence capture on work items is in scope from P1 as optional metadata — automation is not.
- Personal injury and KADOE are out of scope.
- Fee-note exceptions for `QDOS`, `QCL`, `AX`, `FW`, `OAK`, and `ALS` are documented in `handover.docx`.
