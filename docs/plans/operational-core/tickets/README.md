# Tickets

Active tickets promoted from generated plan packs belong here.

Every ticket must include:

- status;
- owner;
- created date;
- last reviewed date;
- source links;
- roadmap milestone;
- dependencies;
- expected documentation/code outputs;
- acceptance criteria;
- verification required;
- pilot/live readiness gate where relevant;
- archive target;
- supersedes/superseded-by fields.

Completed tickets move to `docs/plans/operational-core/archived_plans/implemented/` with an implemented-state block.

Start with `docs/plans/operational-core/tickets/backlog_index.md`.
